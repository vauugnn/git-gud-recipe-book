🍪 Browser Cookies — What They Are & How to Use Them

"Ingredients" (Key Cookie Concepts)

  • name=value — the basic piece of data stored
  • domain — which website the cookie belongs to
  • path — which parts of the site can see the cookie
  • expires / max-age — how long the cookie lasts
  • secure — whether the cookie is only sent over HTTPS
  • HttpOnly — whether the cookie is inaccessible to JavaScript (helps security)
  • SameSite — whether the cookie is restricted to same-site or cross-site contexts

Instructions — How to Use Cookies (Basic Examples)

  1. Decide what data to store (e.g. session token, user preference).
  2. Set the cookie in the backend or via JavaScript. E.g. in JavaScript: 
    
document.cookie = "theme=dark; max-age=31536000; path=/; secure; samesite=strict";
  3. Ensure proper domain and path to match your site structure.
  4. Use "expires" or "max-age" to control how long the cookie stays valid.
  5. For sensitive cookies (login tokens), use HttpOnly and secure flags.
  6. On subsequent page loads, the browser sends cookies matching domain/path — backend reads them to authenticate, personalize, etc.
  7. If needed, delete a cookie by setting its expiration in the past, e.g.: 
    
document.cookie = "theme=dark; max-age=0; path=/";

Pro Tips & Security Notes

  • Always use secure cookies on HTTPS sites to prevent eavesdropping.
  • Set HttpOnly on cookies that shouldn't be accessed by JavaScript (like session data).
  • Use SameSite attributes to reduce the risk of CSRF attacks.
  • Be careful with third-party cookies; many browsers restrict them or block them by default. Clean up or limit usage when possible.
